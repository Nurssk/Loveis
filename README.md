# BirGe — командные покупки 🛒👥

MVP мобильного приложения для **коллективных (командных) покупок**. Пользователи объединяются в команды и получают скидку **5% за каждого участника** (до 30%). Построено на **Expo + React Native + TypeScript + Expo Router**.

> Хакатон-MVP. SMS, SIM/eSIM, оплата и интеграции с маркетплейсами — **симуляция**. Реальный бэкенд не требуется, все данные хранятся локально (AsyncStorage).

---

## ✨ Возможности

- 📱 Вход по номеру телефона (+7) и паролю
- 🔐 Подтверждение по СМС (демо-код `1234`) + концепт SIM/eSIM ID
- 🎯 Выбор интересов и персональная лента рекомендаций
- 🔎 Поиск и фильтрация товаров по категориям
- 👥 Создание команды / присоединение по коду + QR-код
- 🛒 Индивидуальная и командная корзины
- 💸 Скидка 5% за участника (максимум 30%), пересчёт в реальном времени
- 💳 Единый mock-чекаут (Kaspi / Halyk / карта) и экран успешного заказа
- 💾 Сохранение профиля, интересов, команды и корзины между запусками

|-----|----------|
| Телефон | любой корректный `+7 7XX XXX XX XX` |
| Пароль | любой из 6+ символов |
| Код из СМС | **`1234`** |
| Коды команд для входа | **`BUY-2026`**, **`SMART-5`**, **`TEAM-KZ`** |

В разделе **Профиль → Демо-панель для жюри** доступны: заполнить корзины, повторить онбординг, очистить корзины, сбросить все данные.

## 💡 Логика командной скидки

Скидка централизована в [`utils/discount.ts`](utils/discount.ts) и не дублируется по экранам:

```ts
discountPercent = Math.min(teamMemberCount * 5, 30); // 5% за участника, максимум 30%
discountAmount  = subtotal * discountPercent / 100;
finalTotal      = subtotal - discountAmount;
```

| Участников | Скидка |
|-----------:|:------:|
| 1 | 5% |
| 2 | 10% |
| 3 | 15% |
| 4 | 20% |
| 5 | 25% |
| 6+ | 30% |

Кнопка **«Добавить тестового участника»** на экране команды мгновенно пересчитывает цену командной корзины.

## 🎬 Сценарий демонстрации (≈3 минуты)

1. **Вход** — ввести `+7 707 123 45 67` и пароль `123456` → «Войти».
2. **СМС** — ввести демо-код `1234` → показать экран «Профиль защищён через SIM/eSIM ID».
3. **Интересы** — выбрать 2–3 категории (напр. Смартфоны, Спорт) → «Показать рекомендации».
4. **Главная** — показать персональную ленту, попробовать поиск/фильтр по категории.
5. **Товар** — открыть товар, обратить внимание на цену команды и экономию → «Добавить в командную корзину» (появится модалка о команде).
6. **Команда** — «Создать команду» (или вступить по коду `SMART-5`) → нажать **«Добавить тестового участника»** 2–3 раза и показать, как растёт скидка (5% → 10% → 15%…).
7. **Корзина** — переключиться на вкладку «Командная», показать пересчёт скидки и экономии.
8. **Оплата** — «Перейти к оплате», ввести адрес, выбрать Kaspi → «Оплатить».
9. **Успех** — показать номер заказа и сумму со скидкой → «Вернуться на главную».
10. Опционально: **Профиль → Демо-панель** — сброс данных для следующего показа.

## 🌐 Web и деплой

Приложение совместимо с Expo Web (адаптивный «телефонный» макет по центру на широких экранах).

**Статический экспорт:**

```bash
npx expo export --platform web   # результат в ./dist
```

**Хостинг (Vercel и др.):** задеплойте папку `dist` как статический сайт.
- Vercel: `npx vercel deploy dist --prod` (или импорт репозитория, output directory = `dist`).
- Любой статический хост: раздайте содержимое `dist` (SPA-fallback на `index.html`).

**Android-превью (EAS):**

```bash
npm i -g eas-cli
eas build -p android --profile preview
```

> ⚠️ Не публикуйте и не деплойте без явного разрешения — проект только подготовлен к деплою.

## ⚠️ Ограничения MVP

- SMS, SIM/eSIM, оплата и маркетплейсы — симуляция (демонстрационные данные).
- Нет реального бэкенда: всё состояние локально на устройстве.
- «Рекомендации» — простое скоринговое сопоставление по интересам/бюджету/городу, не настоящий ИИ.
- Изображения товаров загружаются из внешнего источника; при сбое показывается иконка-заглушка категории.
- Локализация: интерфейс на русском, строки структурированы для будущего добавления KZ/EN.
